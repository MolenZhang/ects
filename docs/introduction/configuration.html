# 文件配置方式

支持从 JSON 或 YAML 配置文件倒入配置到 ETCD。

# JSON 配置文件

将如下配置保存到 ects.json：

{
  "database": {
    "host": "localhost",
    "port": 3306,
    "name": "ects",
    "user": "root",
    "pass": "PASSWORD",
    "char": "utf8mb4"
  },
  "auth": {
    "secret": "SECRET",
    "ttl": 86400
  },
  "etcd": {
    "killer": "/ects/killer",
    "locker": "/ects/locker",
    "service": "/ects/nodes",
    "pipeline": "/ects/pipelines",
    "config": "/ects/config",
    "endpoints": [
      "localhost:2379"
    ],
    "timeout": 5
  }
}

# YAML 配置文件

将如下配置保存到 ects.yaml：

database:
  host: localhost
  port: 3306
  name: ects
  user: root
  pass: PASSWORD
  char: utf8mb4
auth:
  secret: SECRET
  ttl: 86400
etcd:
  killer: /ects/killer
  locker: /ects/locker
  service: /ects/service
  pipeline: /ects/pipeline
  config: /ects/config
  endpoints:
    - localhost:2379
  timeout: 5
